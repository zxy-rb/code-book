### 2025.12.14  思维+找规律

[E - Minimum Swap](https://atcoder.jp/contests/abc436/tasks/abc436_e)

#### 题目大意

给定一个非升序的 1~N 排列 P，通过交换任意两个位置（i<j）的元素将其排序为升序。设 K 为最少交换次数，求**属于最优操作序列的首次交换的数量**（即首次交换后，剩余交换次数为 K-1 的操作数）。

#### 核心思路：排列的循环分解

排列的**最少交换次数**与它的**循环节结构**密切相关：

1. **循环节定义**：对于排列 P，若元素 i 最终要到位置 P [i]（0-based），则可将排列分解为若干个 “循环”。例如排列`[2,3,1]`（1-based）转成 0-based 是`[1,2,0]`，其循环为`0→1→2→0`（长度 3）。

2. **最少交换次数公式**：设循环节个数为 C，则最少交换次数 K = N - C（每个长度为 L 的循环需要 L-1 次交换，总交换次数为\(\sum (L-1) = N - C\)）。

3. 合法首次交换的条件

   ：

   - 若交换**同一循环内**的两个元素，会将该循环拆分为 2 个循环，循环节个数 C 增加 1，最少交换次数变为 K-1（符合 “最优序列首次操作” 的要求）。
   - 若交换**不同循环内**的两个元素，会将两个循环合并为 1 个循环，循环节个数 C 减少 1，最少交换次数变为 K+1（不符合要求）。

#### 计算合法操作数

对于每个长度为 L 的循环，其内部满足 i<j 的交换对数为组合数$\binom{L}{2} = \frac{L \times (L-1)}{2})$。所有循环的该值之和即为答案。

所有循环的$\frac{L \times (L-1)}{2}$之和可以通过公式简化： $$ \sum_{循环} \frac{L \times (L-1)}{2} = \frac{\sum (L^2) - \sum L}{2} = \frac{\sum (L^2) - N}{2} $$ （因为所有循环的长度和$\sum L = N$）

#### 代码逻辑



1. **排列转 0-based**：将输入的 1~N 排列转为 0~N-1 的索引（方便数组操作）。

2. 循环分解

   ：    

   - 遍历每个位置，若未被处理（`ans[i] == 0`），则找到该位置所在的循环，计算循环长度`count`。
   - 标记该循环内所有位置的循环长度（存入`ans`数组）。

3. 计算结果

   ：

   - 累加`ans`数组（得到$\sum L^2$）。
   - 代入公式$\frac{\sum L^2 - N}{2}$得到合法首次交换的数量。

#### 复杂度分析

- 时间复杂度：\(O(N)\)（仅遍历排列两次，一次计算循环长度，一次标记循环）。
- 空间复杂度：\(O(N)\)（存储排列和循环长度数组）。

### 2025.12.15 树状数组+累加贡献

[F - Starry Landscape Photo](https://atcoder.jp/contests/abc436/tasks/abc436_f)

1. **树状数组（Fenwick Tree）**：
   - 用于**动态统计已处理的星的位置分布**（已处理的星是 “比当前星暗” 的星）；
   - `add(p, 1)`：将亮度为x的星的位置p标记为 “已处理”；
   - `query(p-1)`：查询p左侧已处理的星的数量（即\(L_x\)）。
2. **统计答案**：
   - 按亮度x从小到大遍历（保证处理x时，已处理的星都是比x暗的）；
   - 计算\(L_x\)（左侧暗星数）和Rx（右侧暗星数）；
   - 累加\((Lx + 1) \times (Rx + 1)\)到答案中（“+1” 表示可以不选某侧的暗星）。

```cpp
int N;
    cin >> N;
    vector<int> B(N + 1);  // B[1..N] 对应从东数第i颗星的亮度
    vector<int> pos(N + 1); // pos[x] 表示亮度为x的星的位置（从东数的序号）
    for (int i = 1; i <= N; ++i) {
        cin >> B[i];
        pos[B[i]] = i;
    }
    
    FenwickTree ft(N);
    long long ans = 0;
    for (int x = 1; x <= N; ++x) {  // 按亮度从小到大遍历（x是当前最亮星）
        int p = pos[x];
        // Lx: p左侧已加入的数的数量（即比x暗且在左侧的星数）
        long long Lx = ft.query(p - 1);
        // Rx: 已加入的总数 - Lx（即比x暗且在右侧的星数）
        long long Rx = (x - 1) - Lx;
        ans += (Lx + 1) * (Rx + 1);
        ft.add(p, 1);  // 将当前星的位置加入树状数组
    }
    cout << ans << endl;
```

```c++
// 代码示例：返回 [low, high] 中的恰好包含 target 个 0 的数字个数
// 比如 digitDP(0, 10, 1) == 2
// 要点：我们统计的是 0 的个数，需要区分【前导零】和【数字中的零】，前导零不能计入，而数字中的零需要计入
long long digitDP(long long low, long long high, int target) {
    string low_s = to_string(low);
    string high_s = to_string(high);
    int n = high_s.size();
    int diff_lh = n - low_s.size();
    vector memo(n, vector<long long>(target + 1, -1));
    auto dfs = [&](this auto&& dfs, int i, int cnt0, bool limit_low, bool limit_high) -> long long {
        if (cnt0 > target) {
            return 0; // 不合法
        }
        if (i == n) {
            return cnt0 == target;
        }
        if (!limit_low && !limit_high && memo[i][cnt0] >= 0) {
            return memo[i][cnt0];
        }
        int lo = limit_low && i >= diff_lh ? low_s[i - diff_lh] - '0' : 0;
        int hi = limit_high ? high_s[i] - '0' : 9;
        long long res = 0;
        int d = lo;
        // 通过 limit_low 和 i 可以判断能否不填数字，无需 is_num 参数
        // 如果前导零不影响答案，去掉这个 if block
        if (limit_low && i < diff_lh) {
            // 不填数字，上界不受约束
            res = dfs(i + 1, 0, true, false);
            d = 1;
        }
        for (; d <= hi; d++) {
            // 统计 0 的个数
            res += dfs(i + 1, cnt0 + (d == 0), limit_low && d == lo, limit_high && d == hi);
            // res %= MOD;
        }
        if (!limit_low && !limit_high) {
            memo[i][cnt0] = res;
        }
        return res;
    };
    return dfs(0, 0, true, true);
}
```

```c++
struct edge {
    int v, w;
    // 排序规则：先按边权升序，权相同则按节点编号升序
    bool operator<(const edge& x) const {
        if (w == x.w) {
            return v < x.v;
        }
        return w < x.w;
    }
};
vector<vector<edge>> graph; // 图邻接表
vector<int> ans;            // 存储最终答案
int n;
void DFS(int u) {
    vector<edge> temp, tt;
    for (edge i : graph[u]) {
        temp.push_back(i);
    }
    sort(temp.begin(), temp.end()); // 对当前节点的出边排序
    
    int t = -1; // 记录当前处理的边权分组
    for (edge i : temp) {
        if (t == -1) {
            t = i.w;
        }
        // 遇到新的边权，处理上一个权值分组的所有子边
        if (t != i.w) {
            graph.push_back(vector<edge>());//构造同全值新节点
            for (edge j : tt) {
                graph.back().push_back(j);
            }
            DFS(graph.size() - 1); // 递归处理该权值分组的子节点
            t = i.w;
            tt.clear();
        }
        // 收集当前节点的所有子边，用于后续同权值分组处理 
        for (edge j : graph[i.v]) {
            tt.push_back(j);
        }
        ans.push_back(i.v); // 前序遍历，记录当前节点
    }
    // 处理最后一个边的情况
    if (!tt.empty()) {
        graph.push_back(vector<edge>());
        for (edge i : tt) {
            graph.back().push_back(i);
        }
        DFS(graph.size() - 1);
    }
}
signed main() {
    ios::sync_with_stdio(0);
    cin.tie(0);
    
    cin >> n;
    graph.resize(n + 1);
    // x是父节点，i是当前节点，y是边权
    for (int i = 1; i <= n; i++) {
        int x, y;
        cin >> x >> y;
        graph[x].push_back({i, y});
    }
    DFS(0); // 从根节点0开始遍历   
    for (int i : ans) {
        cout << i << ' ';
    }
    return 0;
}
```

#### 情况 2：`k` 是偶数

需要逐位（从高位到低位）构造，维护两个集合：

- `T`（紧约束）：当前元素的二进制前缀与 `n` 完全一致；
- `L`（松约束）：当前元素的二进制前缀已小于 `n` 的前缀（后续位可以任意取 `1`）。

对每一位 `b`（从高位到低位）：

1. **若 `n` 的第 `b` 位是 `1`**：
   - 要求数组中第 `b` 位为 `1` 的元素个数是**奇数**。
   - 为了最大化和，尽可能多取 `1`：取 `k-1` 个 `1`（因为 `k` 是偶数，`k-1` 是奇数）。
   - 选择一个元素（优先从 `T` 中选）设为 `0`，该元素进入 `L`（后续位可自由取 `1`）。
2. **若 `n` 的第 `b` 位是 `0`**：
   - 要求数组中第 `b` 位为 `1` 的元素个数是**偶数**。

### 2026.1.6 主席树+二分

[Problem - E - Codeforces](https://codeforces.com/gym/105161/problem/E)

### 算法说明

1. **预处理**：对于每个位置 i，生成其操作过程中所有可能的值（即 ai,⌊a~i~/2⌋,⌊a~i~/4⌋,…,0）。这些值视为二维点 (i,y)。
2. **主席树构建**：以位置 i为版本，值 y为权值，建立可持久化线段树。每个版本 `root[i]`维护区间 [1,i]内所有点的 y值分布。
3. **查询处理**：对于每个询问 (l,r,k)，二分答案 ans。利用主席树快速查询区间 [l,r]中 y>ans的点数（即所需操作次数）。若点数 ≤k，说明最终最大值不超过 ans，缩小上界；否则增大下界。最终二分结果即为答案。



```c++
struct node {
    int ch[2], s;
} tr[N * 400];  // 需要更大的节点数
// 修改插入函数，不建空树
void insert(int x, int &y, int l, int r, int v) {
    y = ++idx;tr[y] = tr[x];tr[y].s++;
    if (l == r) return;
    int mid = (l + r) >> 1;
    if (v <= mid) insert(lc(x), lc(y), l, mid, v);
    else insert(rc(x), rc(y), mid + 1, r, v);
}
// 查询值在[L, R]范围内的点数
int query_range(int x, int y, int l, int r, int L, int R) {
    if (!y || L > R) return 0;
    if (L <= l && r <= R) return tr[y].s - tr[x].s;
    int mid = (l + r) >> 1, res = 0;
    if (L <= mid) res += query_range(lc(x), lc(y), l, mid, L, R);
    if (R > mid) res += query_range(rc(x), rc(y), mid + 1, r, L, R);
    return res;
}
```

	给定排列 p₁,p₂,p₃,…,pₙ，定义 Aᵢ表示在 pᵢ左侧并比 pᵢ小的数字个数，Bᵢ表示在 pᵢ右侧并比 pᵢ小的数字个数，Cᵢ = min (Aᵢ,Bᵢ)。现在给定多个操作 (l,r)，求每个操作，交换 (pₗ,pᵣ) 后的∑Cᵢ。

​	Bᵢ = pᵢ - 1 - Aᵢ可以 O (1) 求得；

​	Cᵢ = min (Aᵢ,Bᵢ) 也是 O (1) 得到的

​	由于每个询问相互独立，那么考虑交换 (pₗ,pᵣ) 操作对 Cᵢ的影响：
​	1.对于 **[1,l),(r,n]** 范围的数字，Cᵢ值一定不影响，因为交换操作均在**单侧进行**；

​	2.对于 pₗ，交换到 r 位置后，Aₗ → Aₗ + 区间 [l,r] 小于 pₗ的数字个数，Bₗ' 仍然可以直接求；

​	3.对于 pᵣ，交换到 l 位置后，Aᵣ → Aᵣ - 区间 [l,r] 小于 pᵣ的数字个数，Bᵣ' 仍然可以直接求；

​	4.如果我们在线询问 (主席树维护)，那么对于 pₗ,pᵣ，实际上可以直接两个 O (logn) 重新求。

​	那么重点是对于 **[l + 1, r - 1]** 区间内的数字的 Cᵢ值变化，如何维护？
​	**1、对于 pₗ ≤ pᵢ ≤ pᵣ，如果 Aᵢ ≤ Bᵢ，则交换后 Aᵢ - 1，Bᵢ + 1，从而 Cᵢ - 1；**

​	**2、对于 pₗ ≥ pᵢ ≥ pᵣ，如果 Aᵢ ≥ Bᵢ，则交换后 Aᵢ + 1，Bᵢ - 1，从而 Cᵢ - 1；**

​	**3、对于 pₗ ≤ pᵢ ≤ pᵣ，如果 Aᵢ - 1 ≥ Bᵢ + 1，Aᵢ ≥ Bᵢ，则交换后 Cᵢ + 1；**

​	**4、对于 pₗ ≥ pᵢ ≥ pᵣ，如果 Aᵢ - 1 ≤ Bᵢ + 1，Aᵢ ≤ Bᵢ，则交换后 Cᵢ + 1**

​	**5、分析得    其他情况 c~i~的值保持不变**

### (★)2026.1.16 数论(进制、位运算)+构造

[Problem - E - Codeforces](https://codeforces.com/gym/104461/problem/E)

按十六进制**位**分开统计**每个数字出现的次数**，然后乘以该数字的能耗，最后求和。

**算法分解**

假设我们要计算 0 到 x-1 的所有数（x 是 32 位整数），我们可以将 x 表示为一个 8 位的十六进制数：

```markdown
x = d7 d6 d5 d4 d3 d2 d1 d0  （每个 di 是 0-15 的十六进制数字）
```

**1. 每一位的贡献分离**

对于十六进制的第 k 位（k=0 表示最低位），我们需要统计在 0 到 x-1 的所有数中，这一位上每个数字（0-15）出现的次数。

**2. 出现次数计算**

对于第 k 位，其权重是 16^k。我们设：

- **高位部分**：`high = x >> (4*(k+1))`（即第 k 位以上的所有位）
- **当前位数字**：`cur = (x >> (4*k)) & 0xF`
- **低位部分**：`low = x & ((1<<(4*k))-1)`（即第 k 位以下的所有位）

**那么在第 k 位上：**

1. **完整循环部分**：高位从 0 到 high-1 时，当前位会完整循环 0-15 的每个数字，每个数字出现 `16^k`次
2. **不完整循环的高位**：当高位正好是 high 时，当前位会从 0 循环到 cur-1
3. **当前位等于 cur 的部分**：当高位是 high 且当前位是 cur 时，低位从 0 到 low-1

```c++
ull compute_cost_sum(ull x) {
    if (x == 0) return 0;
    ull ans = 0;
    for (int k = 0; k < 8; k++) {            // 8 个十六进制位
        ull k16 = 1ULL << (4 * k);           // 16^k
        ull high = x >> (4 * (k + 1));       // 高位部分
        int cur = (x >> (4 * k)) & 0xF;       // 当前位的数字
        ull low_part = x & (k16 - 1);        // 低位部分

        ans += high * k16 * prefix[16];            // 完整循环的贡献
        ans += k16 * prefix[cur];             // 当前位小于 cur 的贡献
        ans += low_part * cost[cur];          // 当前位等于 cur 的贡献
    }
    return ans;
}
```

### 核心思路：贡献法

我们不去计算每个子数组的权值，而是计算数组中的**每一个元素 $a[p]$ 对最终总和贡献了多少次**。

1. **判定条件**：一个元素 $a[p]$ 会对某个前缀的“不同元素个数”贡献 $1$，当且仅当它是该前缀中**第一次出现**的该数值。
2. **贡献范围**：假设当前元素是 $a[p]$，它上一次出现的位置是 $prev[p]$（如果没有出现过则为 $0$）。
   - 对于任何起始位置 $l$ 满足 $prev[p] < l \le p$ 的子数组，从下标 $p$ 开始，该元素 $a[p]$ 都是该数值的“首次出现”。
   - 这意味着，对于所有以 $l$ 开头、且包含 $p$ 的子数组 $a[l \dots r]$，在统计其前缀 $a[l \dots k]$（其中 $k \ge p$）的权值时，$a[p]$ 都会贡献 $1$。
3. **数学公式**：
   - 左边界 $l$ 的取值范围有 $p - prev[p]$ 种。
   - 对于固定的 $l$ 和 $p$，右边界 $r$ 可以在 $[p, n]$ 之间取值。
   - 对于每个确定的子数组 $a[l \dots r]$，$a[p]$ 贡献的次数等于它作为“首次出现元素”存在的前缀数量，即 $k \in [p, r]$，共 $(r - p + 1)$ 次。

```c++
int dfs(int u, int p) {
    vector<int> dp;
    for (int v : e[u]) {
        if (v != p) { // 跳过父节点
            dp.push_back(dfs(v, u));
        }
    }
    sort(dp.rbegin(), dp.rend());
    int up = 0, degree = e[u].size(); // 当前节点的度数
    if (degree >= 4) {
        up = dp[0] + 1;
        // 取前两个最大的子路径，计算以u为中点的路径长度
        ans = max(ans, dp[0] + dp[1] + 1);
    } else if (degree == 3) {
        up = 1;
        ans = max(ans, dp[0] + 1);
    }
    return up;
}
```

### 2026.3.7数论（分解质因数）

[Problem - E - Codeforces](https://codeforces.com/contest/2200/problem/E)

如果数组 a 是非递减的，Bob 立即获胜。否则：

如果数组 a 中存在一个元素 x，它能被两个不同的质数 p<q（**不是说q*p=x,q是任何质数**） 整除，那么 Alice 可以通过将 x 替换为 p/x 和 p 来确保自己获胜。由于 p/x 是 q 的倍数(**根据质因数分解过程，p/x一定是后面分解来的质数**)，总会有一个 q 的倍数出现在 p 之前且大于 p，因此数组将永远无法变为非递减的。

如果不存在这样的元素，那么数组 a 中的所有元素要么是 1，要么是形如 pe 的质数幂（其中 p 是某个质数）。定义一个数组 b，其中当 ai=1 时 bi=1，否则 bi 是 ai 唯一的质因数。如果 bi 是非递减的，那么 Bob 会获胜（因为当没有可行操作时，a 也会变为非递减的）。否则，必然存在某个 i 使得 bi−1>bi。如果 e>1，Alice 可以将 ai=pe 替换为 p 和 pe−1。通过与上一段类似的逻辑，这会使得 Bob 无法获胜，因此 Alice 会获胜。
